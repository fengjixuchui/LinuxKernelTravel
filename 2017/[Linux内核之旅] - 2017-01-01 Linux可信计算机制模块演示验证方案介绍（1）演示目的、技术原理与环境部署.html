3.1 演示验证方案

3.1.1 演示目的

        可信计算是以TPM设备作为基础的，TPM设备是可信计算平台的出发点，linux-2.6.12开始以已经支持TPM设备，此验证方案为了验证linux-3.5.4已经能够很好支持TPM设备。Linxu系统可以基于TPM设备，构建可信计算平台，实现封装存储、远程证明等可信计算技术。


3.1.2 技术原理

    使用tpm_emulaor软件包模拟TPM芯片，在Linux系统中搭建一个具有TPM设备的环境。在此环境中以TPM设备为基础可以实现可信计算相关技术，构建可信计算平台。


3.1.3 环境部署

    此处使用tpm_emulator 这款TPM芯片模拟器，它可以模拟TPM芯片的基本功能。使用它以及其他软件包可以搭建一个tpm_emulator环境，达到使用TPM设备的目的。下面是在ubuntu10.04中安装linux-3.5.4内核搭建tpm_emulator 环境的过程。

(1) 安装linux-3.5.4内核

在网站www.kernel.org网站下载linux-3.5.4内核源码压缩包，解压后进入源码解压目录linux-3.5.4。

a. 运行命令：make mrproper，清除原有的配置文件。

b. 运行命令：make menuconfig，用于配置内核信息。在ubuntu10.04下运行此命令时会提示缺少libncurses5-dev包，使用命令sudo  apt-get install  libncurses5-dev 命令安装。再次运行命令 make menuconfig ，进入配置文件选中如下参数：

Device Driver->

Character Device->

[*] TPM Hardware Support->

[*] TPM Interface Specification 1.2 Interface


在内核配置文件中y表示编译进内核，n表示不编译进内核，m表示已模块方式的编译。

a. 运行命令：make bzImage，编译生成可执行的内核引导文件；

b. 运行命令：make modules ,编译生成相应模块；

c. 运行命令：sudo make modules_install，安装内核模块；

d. 运行命令：sudo make install ，安装内核映像。

e. 运行命令: sudo  update-grub2，更新grub，使新编译的内核加入到grub配置文件/boot/grub/gurb.cfg 中。

操作完成后重启系统，进入系统后运行命令：uname –a 可看见系统变成 3.5.4而不是以前的2.6.32-24。ubuntu 10.04 开机没有启动项选择界面，可修改grub 的配置文件/boot/grub/grub.cfg 中的 “timeout=  ”修改启动选择项界面停留时间。

(2) Gmplib库安装

Gmplib库，提供一些密码学算法的支持，在网站http://gmplib.org/可下载到GMPLIB库源码包，最新版本gmp-5.0.5。在安装gmp-5.0.5之前，需要安装如下软件包：m4、libtool、g++。通过apt-get install 安装如上包。安装完成后，进入gmp-5.0.5源码目录。 执行./configure、 make 、make check、 sudo make install 安装gmplib 库即可。

 (3) tpm_emulator和tpm_tools安装

Tpm_emulator模拟tpm芯片，相当于一个虚拟的芯片，可以在网站https://developer.berlios.de/project/showfiles.php?group_id=2491下载，最新版本0.7.4。

解压tpm_emulator-0.7.4源码目录，在解压包内没有configure 文件，所以是不可能执行./configure 命令，但是有cmakelist 文件,所以用cmake 方式进行安装。但要提前安装cmake编译环境，使用sudo apt-get install cmake安装。下来在tpm_emulator-0.7.2文件夹下面建立build 文件夹，tpm_emulator软件将安装在这里。然后执行cmake ../ 命令，会在文件夹build里多出了makefile 然后执行make 、sudo make install。此时，tpm_emulator安装成功。初始化tpm模拟器：tpmd deactivated；kekkall tpmd；tpmd clear,启动TPM模拟器： modprobe tpmd_dev（该命令执行完后，如果出现FATAL: Module tpmd_dev not found，则先运行depmod –a。如果出现failed：address already in use 时，执行rm    /var/run/tpm/tpmd_socket:0 ）；tpmd  -f  -d （tpmd -h查看启动参数） 。

在网站http://sourceforge.net/projects/trousers/files/tpm-tools/下载tpm_tools源码包，最新版为1.3.8，下载完成后解压，进入软件解压目录，运行 ./configure 、make 、make install ，即可安装。在ubuntu10.04的新立得软件中心中找到tpm_tools相关软件包：tpm-tools、tpm-tools-dbg、libtpm-unseal0、libtpm-unseal-dev安装。

(4) trousers安装

trousers软件包提供一些tpm的API函数，在网站http://sourceforge.net/project/showfiles.php?group_id=126012可下载，最新版本0.3.10。

下载源码文件解压，进入解压目录，运行sh ./configure 命令进行环境的检查，会提示缺openssl 软件包，使用sudo apt-get install openssl安装得到新立得软件包，然后搜索openssl ，会发现有libengine-tpm-openssl 、libcurl4-openssl-dev等，其软件是为TPM模块的openssl 的引擎，右击标记，相关联的也要允许标记，然后应用安装。反正要注意查看./configure 中的错误信息，逐一将缺少的软件包装上。下来安装通过apt-get install安装autoconf包，安装完成后执行命令：autorefconf –fiv,下来make 、sudo make install 就可以安装trousers包了。
