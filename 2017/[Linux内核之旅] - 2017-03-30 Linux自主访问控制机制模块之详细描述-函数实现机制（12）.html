acl_permission_check()函数

由上面的分析可以看到generic_permission()函数实际上是通过调用acl_permission_check()函数使用ACL机制来进行权限的检查，该函数定义在fs/namei.c中，函数头如下所示：


static int acl_permission_check(struct inode *inode, int mask)


如上所示，acl_permission_check()函数包含2个参数，其含义和generic_permission()函数相同，这里不再赘述。

函数执行步骤分析：

如图2-26所示，下面结合源码对该函数执行步骤进行分析：

①　调用uid_eq()函数判断进程的fsuid和目标文件的uid是否相等：

a. 若相等，表示当前进程可以访问该文件，于是将表示传统文件模式权限位的变量左移6位，即获得文件所有者对该文件的权限。

b. 如果不相等，首先调用IS_POSIX()宏判断指定的inode所属的文件系统是否支持扩展属性，若支持且该文件所属用户组中的其他用户对该文件具有读写执行权限，则调用check_acl()函数依据ACL机制对文件的访问权限进行检查。然后调用in_group_p()函数判断目标文件所属的用户组ID是否在当前进程的fsgid或附加组中，若是，则将传统文件权限模式位左移3位获得文件所属用户组中的用户对文件的访问权限。

②　判断进程是否具有指定的访问权限，若有，返回0，反之返回-EACCES。
