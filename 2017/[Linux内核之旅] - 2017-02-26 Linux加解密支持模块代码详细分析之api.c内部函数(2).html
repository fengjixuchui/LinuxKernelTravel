3.__crypto_alloc_tfm

函数原型：struct crypto_tfm *__crypto_alloc_tfm(struct crypto_alg *alg, u32 type,u32 mask);

函数功能：算法对象分配，分配成功则返回内核空间地址。

函数参数：alg 算法注册名称 ；type 算法类型； mask 算法掩码。

函数返回值：返回crypto_tfm 对象。

执行流程：

（1）获取算法对象的大小，算法对象大小是由结构体crypto_tfm大小与算法上下文大小之和。

（2）为第1步获得的结果分配内核空间并初始化空间。然后判断空间是否分配成功，如果分配失败，则跳转到第7步。

（3）调用crypto_init_ops初始化算法操作，然后对结果err进行判断，如果获得结果非0，跳转到第6步。

（4）对算法及算法对象进行初始化操作，并判断算法对象是否退出，如果退出失败，则跳转到第5步，否则跳转到第8步。

（5）算法对像的退出操作。

（6）如果第3步结果err等于-EAGAIN,则调用函数crypto_shoot_alg杀死算法。否则释放掉算法对象空间。

（7）返回第3步结果的错误结果给tfm变量。

（8）返回tfm变量。

4.crypto_alloc_base

函数原型：struct crypto_tfm *crypto_alloc_base(const char *alg_name, u32 type, u32 mask);

函数功能：算法对象的分配，此函数不能用于新算法类型。

函数参数：alg_name 算法名称；type 算法类型； mask 算法掩码。

函数返回值：返回crypto_tfm对象 。

执行流程：

crypto_alloc_base算法对象分配主要通过循环实现，在循环内部执行步骤如下：

（1）通过调用函数crypto_alg_mod_lookup在算法模块中查找到算法alg。如果查找失败，返回错误结果并跳转到第4步。

（2）调用__crypto_alloc_tfm为算法alg分配对象tfm，如果分配成功，则返回tfm。

（3）算法模块释放，且第2步分配失败返回失败结果。

（4）第3步结果如果为-EAGAIN,则跳出循环，否则检查当前进程是否有信号处理，如果有，则返回-EINTR并跳出循环。

  循环结束返回err的错误结果。
