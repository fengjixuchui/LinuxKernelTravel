2 扩展属性相关操作在ext4中的实现

前面介绍过，内核根据扩展属性所属的命名空间对扩展属性进行不同的处理，而每个命名空间对扩展属性的处理函数存放在一个名为xattr_handler的结构体中。在ext4中为了提供编码的效率，采用一个名为ext4_xattr_handler_map的指针数组存放了相应命名空间的处理函数集。

static const struct xattr_handler *ext4_xattr_handler_map[] = {

       [EXT4_XATTR_INDEX_USER] = &ext4_xattr_user_handler,

       #ifdef CONFIG_EXT4_FS_POSIX_ACL

      [EXT4_XATTR_INDEX_POSIX_ACL_ACCESS]= &ext4_xattr_acl_access_handler，                                                      [EXT4_XATTR_INDEX_POSIX_ACL_DEFAULT]=&ext4_xattr_acl_default_handler,       #endif

    [EXT4_XATTR_INDEX_TRUSTED] = &ext4_xattr_trusted_handler,

     #ifdef CONFIG_EXT4_FS_SECURITY

    [EXT4_XATTR_INDEX_SECURITY] = &ext4_xattr_security_handler,

     #endif

如上所示，对于各命名空间的处理函数，其实现基本相同或相似，因此下面主要针对用于实现访问控制的ACL进行分析，其处理函数集的定义如下所示：

const struct xattr_handler ext4_xattr_acl_access_handler = {

        .prefix = POSIX_ACL_XATTR_ACCESS,

        .flags  = ACL_TYPE_ACCESS,

        .list   = ext4_xattr_list_acl_access,

        .get    = ext4_xattr_get_acl,

        .set    = ext4_xattr_set_acl,

};


下面主要针对属性的设置函数和获取函数进行分析。

(1) 获取扩展属性

对于用于访问控制的ACL而言，扩展属性的获取函数为ext4_xattr_get_acl()，该函数定义在fs/ext4/acl.c中，函数头如下所示：

static int ext4_xattr_get_acl(struct dentry *dentry, const char *name, void *buffer,

                       size_t size, int type)

    该函数有五个参数：dentry表示目标文件对应的目录项；name表示要获取的扩展属性的名字；buffer表示用于存储扩展属性值的存储空间；size表示存储扩展属性的存储空间的长度；type表示要处理的ACL 的类型。


图2-7 ext4_xattr_get_acl()函数调用流程图

如图2-7所示，下面结合源码对该函数的执行过程进行说明：

①调用test_opt()宏来判断目标文件所在的文件系统对应的超级块是否设置了EXT4_MOUNT_POSIX_ACL标志，若没有，则该函数直接返回-EINVAL。

②调用ext4_get_acl()函数获取指定目录项的指定类型的ACL，对于该函数，下面会进行详细介绍。

③调用posix_acl_to_xattr()函数将内存形式的访问控制列表转换为相应的磁盘表示形式，并将相应的结果存放到buffer指向的缓冲区中。

④调用posix_acl_release()函数释放存放文件指定类型的访问控制类表的存储空间。

⑤结束并返回。 

ext4_get_acl()函数是索引节点操作集中get_acl()钩子在ext4文件系统中的具体实现函数，其定义在fs/ext4/acl.c中，函数头如下所示：

 struct posix_acl *ext4_get_acl(struct inode *inode, int type)

 

该函数包含两个参数：inode表示要操作的目标文件；type表示访问控制列表的类型，其函数调用流程图如图2-8所示：


图2-8 ext4_get_acl()函数调用流程图

下面结合源码对ext4_get_acl()函数的执行步骤进行描述：

①调用test_opt()宏来判断目标文件所在的文件系统对应的超级块是否设置了EXT4_MOUNT_POSIX_ACL标志，如果没有则直接返回NULL。其中EXT4_MOUNT_POSIX_ACL标志表示ext4文件系统在挂载时是否支持posix ACL。

②调用get_cached_acl()函数来根据指定的ACL类型获取索引节点的ACL属性。如果结果等于ACL_NOT_CACHED，即表示ACL没有缓存，则ext4_get_acl()直接返回。

③调用ext4_xattr_get()函数来获取扩展属性的长度，并将结果存放在retval中。

④如果扩展属性的长度retval大于0，则首先调用kmalloc()函数来为扩展属性值申请相应的存储空间，然后再次调用ext4_xattr_get()函数来获取扩展属性的值，并将函数返回值存放到retval中。

⑤如果retval大于0，表示扩展属性已成功拷贝到缓冲区，此时调用ext4_acl_from_disk()函数将ACL从磁盘表示形式转换为内存表示形式，并将结果保存在acl中。反之判断retval是否为ENODATA 或ENOSYS，若是，则将acl置为NULL；否则，使用ERR_PTR()函数宏的相应的错误。

⑥调用kfree()函数释放申请的用于存放扩展属性的内存空间。

⑦判断acl是否存在，若是，调用set_cached_acl()函数根据acl重设文件的指定类型的ACL属性。

⑧结束并返回acl。

对于get_cached_acl()和set_cached_acl()函数，详见2.3.2.2小节，这里不再赘述。下面重点针对ext4_xattr_get()函数从源码上进行分析，该函数根据缓冲区存在与否可以完成两个功能：当缓冲区为空时，该函数用来计算并返回属性值的长度；当缓冲区非空时，则将扩展属性的值拷贝到缓冲区中。该函数定义在fs/ext4/xattr.c中，函数头如下所示：

 int ext4_xattr_get(struct inode *inode, int name_index, const char *name,

                 void *buffer, size_t buffer_size)

如上所示，该函数包含5个参数：inode表示要操作的目标文件；name_index指向扩展属性所属命名空间的处理程序的索引；name表示要获得的扩展属性的名字；buffer表示用于存储扩展属性值的存储空间；buffer_size表示存储空间的长度。


图2-9 ext4_xattr_get()函数调用流程图

如图2-9所示，该函数实际上是对ext4_xattr_ibody_get()和ext4_xattr_block_get()的封装。首先调用ext4_xattr_ibody_get()函数从文件对应的索引节点中的空闲空间中查找指定的扩展属性；如果没有找到，再调用ext4_xattr_block_set()函数从外部的数据块中查找指定的扩展属性。下面分别对这两个函数进行分析：

v ext4_xattr_ibody_get()

ext4_xattr_ibody_get()用于在索引节点的空闲空间中查找指定的扩展属性，成功时，返回扩展属性值的长度。其定义在fs/ext4/xattr.c中，函数头如下所示：

  static int ext4_xattr_ibody_get(struct inode *inode, int name_index, const char *name,void *buffer, size_t buffer_size)

如上所示，该函数包含5个参数，其中各参数的含义和ext4_xattr_get()的参数相同，所以这里不再赘述。如图2-10所示，下面结合源码对ext4_xattr_ibody_get()的执行步骤进行描述：

    ①调用ext4_get_inode_loc()函数来根据指定的索引节点对ext4_iloc结构体进行设置，ext4_iloc结构体用于描述索引节点在磁盘和内存中的位置。

   ②调用ext4_raw_inode()函数来获得目标索引节点对应的磁盘索引节点。

   ③调用IHDR()宏获取存放在索引节点空闲空间内的扩展属性的首部。

   ④调用IFIRST()宏获取索引节点空闲空间内存放的第一个扩展属性。

   ⑤调用ext4_xattr_check_names()函数判断扩展属性项是否位于inode的空闲空间内，若是，返回0；反之，ext4_xattr_get()调用brelse()函数将磁盘索引节点对应的缓冲头的引用计数减1，然后直接返回。

    对于ext4_xattr_check_names()函数，其通过一个循环依次判断各个扩展属性项是否位于inode的空闲空间内。该函数调用了两个重要的宏：IS_LSAT_ENTRY()和EXT4_XATTR_NEXT()。其中IS_LSAT_ENTRY()宏用于判断当前扩展属性项是否位于扩展属性列表中的最后一项；EXT4_XATTR_NEXT()宏用于计算相对当前扩展属性项的下一个扩展属性项的地址，EXT4_XATTR_NEXT()中调用了EXT4_XATTR_LEN()宏，该宏用于计算一个扩展属性项的长度。

    ⑥调用ext4_xattr_find_entry()函数来检查指定的属性是否位于属性列表中。如果没有，则ext4_xattr_get()调用brelse()函数将磁盘索引节点对应的缓冲头的引用计数减1，然后直接返回。

    对于ext4_xattr_find_entry()函数，其采用循环依次遍历扩展属性列表中的每个扩展属性表项。针对每一个扩展属性表项，其先判断指定命名空间的处理程序的索引是否相等，若是再判断属性名的长度是否相等，若是，最后在判断属性名是否相等，若是，则该扩展属性项即为需要的。找到目标扩展属性项之后，先调用ext4_xattr_check_entry()对该属性向进行有效性检查，若未通过检查，则返回错误，反之则根据查找结果进行返回。

   ⑦如果buffer存在，则调用memcpy()函数将扩展属性值拷贝到指定的缓冲区。

   ⑧调用brelse()函数将磁盘索引节点对应的缓冲头的引用计数减1。

   ⑨结束并返回。


图2-10 ext4_xattr_ibody_get()函数调用流程图

v ext4_xattr_block_get()

ext4_xattr_block_get()函数用于在扩展属性所在的外部数据块中查找指定的扩展属性，其定义在fs/ext4/xattr.c中，函数头如下所示：

tatic int ext4_xattr_block_get(struct inode *inode, int name_index, const char *name,

                           void *buffer, size_t buffer_size)

如上所示，该函数也包含5个参数，其含义和和ext4_xattr_get()的参数相同，所以这里不再赘述。对于该函数，其函数调用流程图如图2-11所示，下面结合源码对该函数的执行步骤进行说明：

①调用EXT4_I()函数来获取目标索引节点对应的内存索引节点ext4_inode_info，然后判断其中的i_file_acl字段是否设置，该字段存放了扩展属性所在的磁盘块号。

②调用sb_bread()函数。该函数只是简单的封装了__bread()函数，后者用于读取一个磁盘块到缓冲区中，并返回该缓冲区所在的缓冲头。

③调用ext4_xattr_check_block()函数来检查目标对象所在的磁盘块所对应的缓冲区是否正确，若不对，则调用EXT4_ERROR_INODE()宏对该错误进行处理。

④调用ext4_xattr_cache_insert()函数在扩展属性所在的缓冲区中创建一个新的表项。

⑤调用BFIRST()宏获取扩展属性表项中的第一项。

⑥调用ext4_xattr_find_extry()函数在扩展属性列表中查找指定的扩展属性，找到的话，返回0。

⑦接着调用le32_to_cpu()函数将磁盘中采用小端格式存储的扩展属性的长度转换为内存格式，并存放到size中。

⑧如果用于存放扩展属性值的缓冲区存在，则使用memcpy()函数将扩展属性值拷贝到相应的缓冲区中。

⑨调用brelse()函数将扩展属性所在的磁盘块对应的缓冲区的头中的引用计数减1。

⑩结束并返回。
