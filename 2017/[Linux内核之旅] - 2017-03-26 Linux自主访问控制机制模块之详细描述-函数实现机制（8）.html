ext4_init_acl()

ext4_init_acl()函数用于初始化一个新文件的ACL，函数头如下所示：


int ext4_init_acl(handle_t *handle, struct inode *inode, struct inode *dir)


该函数包含三个参数：handle表示用于日志处理的原子更新操作；inode指向新建文件的索引节点；dir表示文件所属目录的索引节点，该函数之所以需要知道新建文件所在目录的inode是因为如果文件所属目录包含默认ACL，那么其包含的内容将会应用到新建的文件，如果目录所在文件系统不支持ACL或没有关联默认的ACL，那么内核只是将进程的当前UMASK设置应用到文件。

ext4_acl_chmod()

ext4_acl_chmod()函数用于在调用chmod命令修改文件的权限时，根据指定的权限来修改目标文件中用于访问控制的ACL，函数头如下所示：


int ext4_acl_chmod(structinode *inode)


该函数只有1个参数：inode表示目标索引节点。对于该函数，其函数调用流程图如图2-19所示：



图2-19 ext4_acl_chmod()函数调用流程图


对于该函数，其调用的函数上面均已介绍过，所以这里不对其进行详细说明，只简单介绍下ext4_journal_start()函数和ext4_journal_stop()函数。这两个函数用于日志处理，因为扩展属性的修改是一个文件系统事务，所以实现时必须采用这两个函数将其包围起来，以确保对其修改的原子执行。而ACL就是通过扩展属性实现的，所以在调用ext4_set_acl()函数修改ACL时，必须使用上述函数将其包围起来以确保ACL修改的原子执行。
