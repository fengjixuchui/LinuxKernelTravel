generic_permission()


下面主要针对generic_permission()函数从源码上进行分析。generic_permission()函数用于posix类文件系统的访问权限的检查，成功时，返回0。其定义在fs/namei.c中，函数头如下所示：


int generic_permission(struct inode *inode, int mask)


该函数有两个参数：inode表示要进行权限检查的目标文件；mask表示要检查的权限。对于该函数，其函数调用流程图如图2-25所示：



下面结合源码对该函数的执行步骤进行说明：

①　调用acl_permission_check()函数根据ACL机制对文件的访问权限进行检查。

②　判断目标文件是不是目录，如果是：

a. 调用inode_capable()函数判断当前进程是否具有CAP_DAC_OVERRIDE能力，若有，则generic_permission()函数直接返回0。

b. 判断mask中是否指定了MAY_WRITE标志，如果没有且当前进程没有CAP_DAC_READ_SEARCH能力，则generic_permission()函数直接返回0。

c. generic_permission()函数结束，并返回-EACCES。

③　如果mask中没有设置MAY_EXEC标志或者目标文件对于所有者和所有者所在组之外的其余用户没有执行的权限，且当前进程具有CAP_DAC_OVERRIDE能力，则generic_permission()函数直接返回0。

④　如果mask只设置了MAY_READ标志，且当前进程具有CAP_DAC_READ_SEARCH能力，则generic_permission()函数直接返回0。

⑤　结束并返回-EACCES。

其中，inode_capable()函数用于判断当前进程是否具有对指定文件的指定能力，如果当前进程在自己的命名空间中有指定的能力且指定的索引节点位于当前用户命名空间或其子命名空间，则返回真。对于generic_permission()函数所要判断的两种能力：CAP_DAC_OVERRIDE和CAP_DAC_READ_SEARCH，如果进程具有CAP_DAC_OVERRIDE能力，表示当该进程访问文件时，忽略对文件的所有DAC访问限制；如果进程具有CAP_DAC_READ_SEARCH能力，表示当该进程访问文件时，忽略对读和搜索的限制。
