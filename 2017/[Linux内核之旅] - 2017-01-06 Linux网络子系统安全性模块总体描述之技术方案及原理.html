1.3 技术方案及原理

查阅Linux相关的安全资料，针对Netfilter防火墙框架和XFRM基于IPsec协议的安全框架进行理论性研究，选取Netfiter下NAT机制和基于XFRM框架的IPsec机制做重点分析，对其原理及源代码的实现进行深入研究。

 

1.3.1 Netfilter相关原理

Netfilter是内核中的一个框架，它可以为内核防火墙模块提供有状态或无状态的包过滤服务，从而实现NAT、IP伪装等，也可以因高级路由或连接状态管理的需要而修改IP头信息 [1]。

NAT，英文Network Address Translation，即网络地址转换。NAT有三种类型：静态地址转换NAT(Static NAT)、动态地址转换NAT(Pooled NAT)、网络地址端口转换NAPT（Port－Level NAT）。

NAT主要的应用场景是：局域网内部的主机要与互联网进行通讯时，就在网关处将内部地址替换成公用地址，使多台计算机共享Internet连接。通过这种方法，可以只申请一个合法IP地址，但却可以把整个局域网中的计算机通过这一个合法IP接入Internet中。NAT屏蔽了内部网络，所有内网的计算机对于公共网络来说是不可见的。NAT网络地址转换主要有两种：SNAT和DNAT。SNAT，英文Source Network Address Translation，即源地址转换。本报告以SNAT分析为主。

SNAT的功能是将IP数据包的源地址转换成另外一个地址。假设内网主机A(192.168.2.8)要和外网主机B(61.132.62.131)通信，A向B发出IP数据包，如果没有SNAT对A主机进行源地址转换，A与B主机的通讯会不正常中断，当路由器将内网的数据包发到公网IP后，公网IP会给内网IP回数据包，从公网上根本就无法看到内网IP，因此内网主机是无法与公网通信的。为了实现数据包的正确发送及返回，网关必须将A的地址转换为一个合法的公网地址，同时为了以后B主机能将数据包发送给A，这个合法的公网地址必须是网关的外网地址，如果是其它公网地址的话，B会把数据包发送到其它网关，而不是A主机所在的网关，A将收不到B发过来的数据包，所以内网主机要上公网就必须要有合法的公网地址，而得到这个地址的方法就是让网关进行SNAT(源地址转换)，将内网地址转换成公网址(一般是网关的外部地址)，所以经常会看到为了让内网用户上公网，必须在Firewall中设置SNAT，俗称IP地址欺骗或伪装。

 

1.3.2 IPsec技术方案

IPsec（IP Security）是Internet的网络层安全协议，是IETF制定的为保证在Internet上传送数据的安全保密性能的框架协议。XFRM框架提供了IPsec的技术实现，为网络通信的双方提供了相对安全的虚拟数据链路，不仅能够抵御各种常见的网络攻击，对于难以防范的重放攻击也能有效杜绝。

IPsec应用最广泛的就是VPN（虚拟专用网），在虚拟专网中，任意两个节点之间的连接并没有传统专网所需的端到端的物理链路，而是利用某种公众网的资源动态组成，IPsec VPN就是利用IPsec技术在Internet上建立的VPN。

IPsec的安全性依赖于Linux中的加密封装协议，最主要的有两种：AH（Authentication Header，认证头协议）、ESP（Encapsulating Security Payload，封装安全负载协议）。这两种协议负责对原来数据包进行重新封装，以达到一定的安全性的要求。前者在原来的IP头后插入AH头标，包的其它部分保持不变，其中AH头标包括：下一头标、净荷长度、SPI(Security Parameters Index，安全参数索引)、序列号、认证数据；后者在原来的IP头后插入ESP头标，并对传输层头标、净荷、ESP尾标进行加密，并在包尾加上ESP认证数据，其中ESP头标包括：SPI、序列号、下一头标等信息。此外，IPsec还有另外一种协议， 即IPCOMP（IP压缩协议）协议，它并不提供安全性保障，主要提供数据压缩功能。

IPsec的应用场景主要有三种，分别为：端到端、端到网关、网关到网关；工作模式主要有两种，一种是传输模式，一种是隧道模式，传输模式只适用于端到端这一种场景，后两种场景则只能使用隧道模式。传输模式在重装封装的时候最外层的网络层头依然是原来的IP头，隧道模式则需要在最外层加入新的IP头标，来指明网关源IP以及目的IP。

IPsec的管理机制主要是依赖于SPD（Security Policy Database，安全策略数据库）和SAD（Security Association Database，安全关联数据库），SPD主要是用来指定作用于到达或者源自特定主机或者网络的数据流的策略；SAD则用来为通信双方的认证/加密算法及其参数、密钥的一致，相互间建立联系。SAD和SPD之间是通过SAID进行关联的。通过查看SPD中的SAID值，可对SAD进行查找，找到该策略项所应该实施的SA。SPD使用选择符来查找，选择符可以是五元组（目的/源地址，协议，目的/源端口号），或其中几个，而SAD则可以通过三元组（SPI、目的/源IP、加密协议）来查找。

双方在建立连接前首先要进行安全参数的协议，即进行SA协商，以此来建立安全通道，这一过程非常关键，为后续的数据传输提供安全保障，协商完毕后双方会分别持有一个SA，形成SA对，并存在双方的SAD中，并会建立起一条安全通道以供数据通信。协商的过程依赖于IKE（Internet Key Exchange，网络密钥交换）协议，IKE的功能包括：

1．协商通信参数，安全特性

2．认证通信对端

3．保护实体

4．用安全的方法产生，交换，建立密钥

5．管理，删除安全关联（SA）

协商分为两个阶段：

第一阶段为建立IKE本身使用的安全信道而相互交换SA（采用ISAKMP）——ISAKMP SA（双向）。

第二阶段利用第一阶段建立的安全信道交换IPSec通信中使用的SA——IPSec SA（单向）。

IPsec应用最广泛的就是VPN（虚拟专用网），在虚拟专网中，任意两个节点之间的连接并没有传统专网所需的端到端的物理链路，而是利用某种公众网的资源动态组成的，IPSec VPN就是利用IPSec技术在Internet上建立的VPN，其应用还是较为广泛的。
