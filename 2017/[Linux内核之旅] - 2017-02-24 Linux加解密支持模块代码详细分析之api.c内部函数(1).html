2.4.2.5.内部函数


文件api.c中涉及主要内部函数的功能, 函数原型、参数、返回值，调用关系，执行流程、流程图等信息如下。

1.__crypto_alg_lookup

函数原型：static struct crypto_alg *__crypto_alg_lookup(const char *name, u32 type,u32 mask);

函数功能：算法查找。

函数参数：name 算法名称；type 算法类型； mask 算法掩码。

函数返回值：返回crypto_alg对象 。

执行流程：

函数__crypto_alg_lookup主要通过遍历链表crypto_alg_list并返回算法q进行处理。

（1）判断算法的状态，算法是否已经死亡或者算法正在死亡，如果算法死亡则跳过遍历中的后续操作继续遍历。

（2）判断算法的标志，如果判定结果为真，则跳过遍历中的后续操作继续遍历。

（3）由q生成的larval算法以及产生larval之后相关的操作是否成功，如果成功则跳过遍历中的后续操作继续遍历。

（4）算法驱动名与传入的算法名不同且算法名对应的优先级大于给定的优先级，则跳过遍历中的后续操作继续遍历。

（5）获取q对应的模块可能性比较小，则跳过遍历中的后续操作继续遍历，并重新设置算法优先级。

（6）针对第4步算法驱动名与传入算法名如果相同，则跳出链表遍历，返回算法alg。

2.crypto_ctxsize

函数原型：static unsigned int crypto_ctxsize(struct crypto_alg *alg, u32 type, u32 mask);

函数功能：获取算法上下文大小。

函数参数：alg 算法注册名称 ；type 算法类型； mask 算法掩码。

函数返回值：返回算法上下文长度。

执行流程：

（1）获取算法对齐掩码和算法对象上下文对齐取反减1相与的结果len。

（2） 对算法类型进行判断，如果判定为真，则返回第一步结果len与算法类型对应上下文结果的和。通过算法不同标志对算法上下文进行处理。当算法标志为CRYPTO_ALG_TYPE_CIPHER时，算法上下文为第1步结果与算法cipher上下文结果之和；当算法标志为CRYPTO_ALG_TYPE_COMPRESS时，算法上下文为第一步结果与算法compress上下文结果之和；其他情况则返回BUG()。
