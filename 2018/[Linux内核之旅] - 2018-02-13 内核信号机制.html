1 信号（signal）

在中断和异常一章，80x86处理器发布了大约20种异常，每一种异常都有其对应的异常向量和异常处理程序。比如向量0对应除法故障，向量15对应页故障。当15号异常发生时，异常处理程序就会向触发该异常的进程发送SIGSEGV信号。所以一句话总结什么是信号：信号是用于通知进程特定事件的发生的一个整数。

信号有两种状态：未决阻塞(pending and blocked)和未决未阻塞(pending and nonblocked)。 
当一个信号产生时，内核通常在进程表中以某种形式设置一个标志。当对信号采取某些动作时（比如执行信号处理程序），称递送(delivery)了一个信号。在信号产生（特定事件发生）和递送的间隙，称信号时未决的。

进程可以选择阻塞信号的递送。如果为进程产生了一个阻塞的信号，而且对该信号的动作是系统默认动作和捕捉该信号，则为该进程将此信号保持为未决状态，直到对该信号解除了阻塞，或者将该信号的动作改为忽略。进程可以通过调用sigpending函数判断那些信号是设置为未决并阻塞的。

每个进程都有一个信号屏蔽字(signal mask)，它规定了进程要阻塞的信号。函数sigprocmask可以设置进程的信号屏蔽字。

内核中共有64种信号，前32种是普通信号(regular signal 0 - 31)后32种是实时信号(rt-signal 32-63)，一个普通信号被发送多次，只有一个被发送到接收进程。实时信号可以被排队发送。内核中并不使用实时信号。

2 与信号有关的数据结构

2.1 task_struct和信号

本节叙述进程描述符和信号的关系，在task_struct中存在几个与信号有关的数据结构： 
signal中包含一个存放共享信号的挂起队列，该结构的类型是struct signal_struct。

2.2 未决信号队列(pending signal queue)


和未决信号队列有关的结构由两个：sigpending结构和sigqueue结构。sigpending是队列的头节点存储有关于信号队列的信息。一个信号位图signal，signal的每一位表示一种信号，通过signal就可以得到队列中到底有哪些信号在排队。 


每个进程有两种信号队列，一个是私有未决信号队列pending，另一个是共享未决信号队列shared_pending。在信号的传递时，优先处理私有队列中的信号。

  11.     struct user_struct *user;

2.3 sigaction结构

sigacition有两个主要字段，sa_handler表明如何处理信号。sa_flags则提供了为信号处理添加了额外的行为，如SA_RESTART(自动重新启动被信号中断的系统调用)。 
sa_handler有三种值，SIG_DFL（执行缺省操作），SIG_IGN（忽略信号），或者信号处理程序的一个指针。

send_signal负责将信号插入到目标进程的pending队列中，该函数接收四个参数；

  24.             q->info.si_pid = current->pid;

  31.             q->info.si_pid = 0;
  32.             q->info.si_uid = 0;
  33.             break;

  35.             copy_siginfo(&q->info, info);
  36.             break;

4 信号的传递(delivery)

信号的传递就是对信号的处理过程。

4.1 信号处理的时机

内核在允许进程恢复在用户态的执行之前，检查thread_info的标志TIF_SIGPENDING的值。每当内核处理完一个中断或异常时，就检查是否存在未决的信号。

4.2 忽略和默认操作

内核调用do_signal()函数处理信号。前面提过，对于每种信号内核有三种处理手段， 忽略信号，执行默认操作，捕捉信号。

do_signal 接收两个参数： 
struct pt_regs regs : regs是内核的硬件上下文，保存一组寄存器值 
sigset_t oldset ：oldset指向信号接收进程的信号屏蔽字

do_signal的核心是重复调用dequeue_signal()函数的死循环，直到私有未决信号队列和共享未决信号队列都没有非阻塞的未决信号时，循环结束。dequeue_signal首先处理私有未决信号队列中的信号然后处理共享队列中的信号。

  1. signr = dequeue_signal(current, mask, info);


et_signal_to_deliver

4.2 捕捉信号handle_signal

信号的捕捉由handle_signal完成，函数由5个参数 
sig: 信号编码 
info: 携带有描述信号的信息 
ka: 携带有对于信号的处理函数 
oldset: 信号屏蔽字 
regs: 内核栈中的硬件上下文（寄存器）

关于信号处理函数的执行有一点需要说明，由于信号处理函数是在用户态下执行，而从内核态切换到用户态时内核栈会被清空。而内核栈中保存有被信号中断的程序的cs和eip。如果内核栈被清空则无法在从内核态中恢复被信号中断的程序的执行。 
解决办法是在内核处理信号时，把保存在内核态堆栈的硬件上下文拷贝到当前进程的用户态堆栈中。 
这样在信号处理末期，就可以从内核栈中恢复被中断程序的CS和EIP，从而恢复程序的正常执行。


  1. tatic void
  2. handle_signal(unsigned long sig, siginfo_t *info, struct k_sigaction *ka,
  3.           sigset_t *oldset, struct pt_regs * regs)
  4. {
  5.     /* Are we from a system call? */
  6.     if (regs->orig_eax >= 0) {
  7.         /* If so, check system call restarting.. */
  8.         switch (regs->eax) {
  9.                 case -ERESTART_RESTARTBLOCK:
  10.             case -ERESTARTNOHAND:
  11.                 regs->eax = -EINTR;
  12.                 break;
  14.             case -ERESTARTSYS:
  15.                 if (!(ka->sa.sa_flags & SA_RESTART)) {
  16.                     regs->eax = -EINTR;
  17.                     break;
  18.                 }
  19.             /* fallthrough */
  20.             case -ERESTARTNOINTR:
  21.                 regs->eax = regs->orig_eax;
  22.                 regs->eip -= 2;
  23.         }
  24.     }
  26.     /* Set up the stack frame */
  27.     if (ka->sa.sa_flags & SA_SIGINFO)
  28.         setup_rt_frame(sig, ka, info, oldset, regs);
  29.     else
  30.         setup_frame(sig, ka, oldset, regs);
  32.     if (!(ka->sa.sa_flags & SA_NODEFER)) {
  33.         spin_lock_irq(&current->sighand->siglock);
  34.         sigorsets(&current->blocked,&current->blocked,&ka->sa.sa_mask);
  35.         sigaddset(&current->blocked,sig);
  36.         recalc_sigpending();
  37.         spin_unlock_irq(&current->sighand->siglock);
  38.     }
  39. }
