 3审计监视处理实例 audit_watch_group

文件系统变化的审计消息是通过审计监视处理实例 audit_watch_group进行处理的,它的操作函数集audit_watch_fsnotify_ops, 操作函数集中的函数 audit_watch_handle_event 处理 fsnotify事件。当文件系统发生变化时, fsinotify 机制hook函数会通过 watch 调用函数 audit_watch_handle_event,将文件系统变化的审计消息通过审计系统写入到 log 文件中。

audit_watch_group由audit_watch_init函数初始化，该函数在内核引导阶段被调用执行。

操作函数集 audit_watch_fsnotify_ops列出如下:

    static const struct fsnotify_ops audit_watch_fsnotify_ops = {

        .should_send_event =    audit_watch_should_send_event,

        .handle_event =         audit_watch_handle_event,

        .free_group_priv =      NULL,

        .freeing_mark =         NULL,

        .free_event_priv =      NULL,

};

其中, 操作函数audit_watch_should_send_event用来转发事件，操作函数audit_watch_handle_event用来处理事件。事件的监视处理过程为：函数audit_watch_handle_event 根据 fsnotify 事件的类型进行事件处理，主要工作为：更新审计规则结构实例上的审计监视数据，调用审计系统接口函数将文件系统变化信息写入到 log 文件中。

（1）audit_watch_handle_event接口功能说明

   函数名：audit_watch_handle_event

   参数：group为处理事件的group,inode_mark为索引节点发生事件时的mark信息，vfsmount_mark为挂载点发生事件时的挂载点信息，event为所发生的事件。

   返回值：如果事件处理成功返回0否则返回错误码。

功能：处理fsnotify所监视到的事件。

（2）函数分析

该函数首先根据发生事件的是文件还是目录，找到正确的inode信息。然后根据不同的事件类型，调用audit_update_watch函数规则的监视信息，比如创建了一个文件，就要更新规则的inode信息。该函数处理流程如下图：
