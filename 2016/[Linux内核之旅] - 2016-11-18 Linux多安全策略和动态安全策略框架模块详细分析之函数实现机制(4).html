(5) 访问向量表

访问向量表用于表示策略中的访问向量规则，表中的每一个节点均对应一条规则。该规则由规则名称、源类型、目标类型、客体类别和权限五部分组成，其中规则名称、源类型、目标类型和客体类别唯一的确定了相应的访问权限。在SELinux中，使用avtab来表示访问向量表，其定义在securit/selinux/ss/avtab.h中，具体定义如下：

   struct avtab {

          struct avtab_node **htable;

          u32 nel;        /* number of elements */

          u32 nslot;      /* number of hash slots */

          u16 mask;       /* mask to compute hash func */

   };

如上所示，代码中已经对各字段的含义进行了解释，这里不再赘述。对于每个访问向量节点，SELinux使用avtab_node结构表示，其定义如下：

struct avtab_node {

          struct avtab_key key;

          struct avtab_datum datum;

          struct avtab_node *next;

   };

各字段含义参见表2-7：

对于avtab_key结构，其用来表示一个访问向量的键值，定义如下：

struct avtab_key {

          u16 source_type;        /* source type */

          u16 target_type;        /* target type */

          u16 target_class;       /* target object class */

  #define AVTAB_ALLOWED           0x0001

  #define AVTAB_AUDITALLOW        0x0002

  #define AVTAB_AUDITDENY         0x0004

  #define AVTAB_AV    (AVTAB_ALLOWED | AVTAB_AUDITALLOW | AVTAB_AUDITDENY)

  #define AVTAB_TRANSITION        0x0010

  #define AVTAB_MEMBER            0x0020

  #define AVTAB_CHANGE            0x0040

  #define AVTAB_TYPE    (AVTAB_TRANSITION | AVTAB_MEMBER | AVTAB_CHANGE)

  #define AVTAB_ENABLED_OLD   0x80000000 /* reserved for used in cond_avtab */

  #define AVTAB_ENABLED           0x8000 /* reserved for used in cond_avtab */

          u16 specified;  /* what field is specified */

  };


如上所示，其中specified字段指定访问向量的规则名称，其余各字段的含义代码中已经进行了解释，这里不再赘述。

对于avctab_datum结构，其用来表示访问向量对应的数据，定义如下：

struct avtab_datum {

           u32 data; /* access vector or type value */

   };

访问向量表的相关操作如表2-8所示，这些函数均定义在security/selinux/ss/avtab.c中。

表2-8 访问向量表相关操作说明

函数名称

函数功能

avtab_hash()

访问向量表对应的哈希函数

avtab_insert_node()

在指定的访问向量表中插入一条访问向量规则，成功执行时返回新插入的节点

avtab_insert()

查找待插入的节点在访问向量表中的位置，然后插入待插入节点，成功执行时返回0

avtab_insert_nonunique()

在指定的位置插入指定的访问向量，其余avtab_insert()函数的区别在于该函数允许同一条访问向量多次插入表中

avtab_search()

在访问向量表中搜索指定的键值对应的访问向量值。成功执行时返回相应的值，反之返回NULL

avtab_search_node()

搜索指定的键值对应的访问向量规则。成功执行时返回相应的表示访问向量的节点，反之返回NULL

avtab_search_node_next()

搜索与指定的访问向量相同的下一个访问向量。成功时返回相应的访问向量，反之返回NULL

avtab_init()

初始化访问向量表

avtab_destroy()

销毁访问向量表

avtab_alloc()

为包含指定数量访问向量的访问向量表h分配相应的存储空间，并初始化该访问向量表。成功时返回0

avtab_hashtab_eval()

计算访问向量中已经使用的位置的数量，最长的链表长度以及链表长度的平方和

avtab_read()

从二进制策略文件中读取访问向量并将其插入访问向量表中，成功时返回0

avtab_write_item()

将一个访问向量规则写入二进制策略文件中

avtab_write()

将访问向量表写入到二进制策略文件中

avtab_cache_init()

初始化访问向量节点缓冲区

avtab_cache_destroy()

销毁访问向量节点缓冲区

avtab_read_item()

从二进制策略文件中读取访问向量并将其存入访问向量表中。该函数根据策略库的版本进行不同的处理
