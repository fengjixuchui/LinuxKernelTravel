2. SID与文件的绑定

在Flask体系结构中，定义了两个与安全策略无关的数据类型来对客体进行标记，分别为安全上下文和安全标识符。当主体创建客体时，对象管理器会将创建客体的主体的SID、相关客体的SID和客体的类型传递给安全服务器，安全服务器在接收到这些数据后对其进行处理，从而为新建的客体生成一个安全上下文并将其转化为SID，然后将其返回给对象管理器，对象管理器获得该SID后将其与相应的客体进行绑定。本节主要以文件系统为例说明文件创建时SID与文件的绑定过程。

在Linux中，文件的创建是通过open()或者creat()系统调用进行的，那么在创建文件时，SELinux中的对象管理器是如何将SID与文件绑定的呢？下面从源码的角度对这个过程进行分析。


图2-20 创建文件时SID与文件绑定的过程调用图

如图2-20所示，其中灰色背景标注的红色文字表示用于文件创建的钩子函数，在ext4文件系统中该函数对应的回调函数为ext4_create()。当调用open()或creat()系统调用创建文件时，它们会按上面的函数调用过程来为新建的文件绑定一个对应的SID，这里以ext4文件系统为例来进行说明。当在ext4文件系统中调用ext4_new_inode()函数为新建的文件创建inode时，其中会调用ext4_init_security()函数来对文件的安全属性进行设置，该函数只是简单了封装了security_inode_init_security()函数。security_inode_init_security()函数用于设置inode对应的扩展属性，该函数定义在security/security.c中，函数头如下所示：

int security_inode_init_security(struct inode *inode, struct inode *dir,

                             const struct qstr *qstr,const initxattrs initxattrs,

 void *fs_data)

该函数有5个参数：inode表示待创建的文件的inode；dir表示待创建的文件所在的目录的inode；qstr表示新建文件路径中的最后一部分内容；initxattrs指向ext4文件系统中将扩展属性写入inode的回调函数；fs_data表示文件系统的相关信息，在调用initxattrs()函数将扩展属性写入inode时使用。对于该函数，其函数调用流程图如图2-21所示，下面结合源码对该函数的执行步骤进行说明：


①　调用IS_PRIVATE()宏判断参数指定的inode是否为文件系统私有的inode，若是，该函数直接返回0。

②　调用memset()函数来对newxattr进行初始化，这里将其设置为0，其中new_xattrs变量存放新建文件的相关扩展属性。

③　判断initxattrs()函数是否设置，若没有，则表示安全模块不希望将安全属性写入文件的inode中，此时直接调用安全模块中的inode_init_security()来为新建的文件生成一个SID，并将其放到inode对应的安全inode中的sid中，然后返回。对于SELinux，该钩子对应的回调函数为selinux_inode_init_security()，对于该函数，下面会详细进行分析。

④　调用安全模块实现的inode_init_security()函数来为新建的文件创建一个SID，并将该SID对应的安全上下文及安全上下文的长度存放指定的参数中。

⑤　调用evm_inode_init_security()函数来初始化文件的security.evm属性。

⑥　调用initxattrs()函数将获取的新建文件的相应安全属性写入到文件的扩展属性中，这里该函数实际对应ext4_initxattrs()函数。对于ext4_initxattrs()函数，其实际上通过调用ext4_xattr_set_handle()函数来将相应的安全属性写入到文件的扩展属性中，至于ext4_xattr_set_handle()函数，详见“Linux自主访问控制机制模块代码分析报告”一文，这里不再赘述。

⑦　通过for循环释放用于存放安全属性名字和值的临时空间。

⑧　结束并返回。


图2-21 security_inode_init_security()函数调用流程图

在SELinux中，inode_init_security()钩子对应的回调函数为selinux_inode_init_security()，该函数用于为新建的文件生成一个SID，并将该SID对应的安全上下文及安全上下文的长度分别存放到参数value及len中。对于该函数，其定义在security/selinux/hooks.c中，函数头如下所示：

static int selinux_inode_init_security(struct inode *inode, struct inode *dir,

                                const struct qstr *qstr, char **name,

                                void **value, size_t *len)


该函数包含6个参数：inode表示新建的inode；dir表示新建文件所在目录的inode；qstr表示新建文件路径中的最后一部分内容；name用来存放安全属性对应的扩展属性的名字；value用来存放相应的安全属性值；len用来存放安全属性的值的长度。对于该函数，其函数调用流程如图2-22所示，下面结合源码对该函数的主要执行步骤进行说明：

①　根据当前进程的主体凭证获取进程的SID及其新建的文件的SID。

②　如果目录所在的文件系统对应的安全超级块结构体中设置了SE_SBINITIALIZED标志，并且该文件系统使用挂载点来对文件进行标记，则将新建文件的SID设置为挂载点的SID。反之判断新建文件的SID及目录所在的文件系统对应的安全超级块结构体，如果新建文件的SID为空或者目录所在的文件系统对应的安全超级块结构体没有设置SE_SBLABELSUPP标志，则调用security_transition_sid()函数为新建的文件创建一个SID。

③　判断目录所在的文件系统对应的安全超级块结构体中是否设置了SE_SBINITIALIZED标志，如果设置了该标志，则根据相关信息设置新建文件的安全inode结构。在设置安全inode结构体中文件的类别时，这里调用了inode_mode_to_security_class()函数，该函数根据新建文件的inode中指定的文件类型来计算文件的客体类别。

④　判断name是否存在，如果存在，则调用kstrdup()函数将SELinux的安全属性对应的扩展属性的名字拷贝到一个空间中，并让name指向该空间。

⑤　判断value和len是否存在，如果两者均存在，则调用security_sid_to_context_force()函数将新建文件的SID转化为对应的安全上下文，并让value指向该安全上下文，len指向该安全上下文的长度。

⑥　结束并返回。
