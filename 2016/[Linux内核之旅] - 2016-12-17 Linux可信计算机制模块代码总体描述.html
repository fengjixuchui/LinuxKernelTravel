1.1 概述

随着计算机技术的不断发展，安全问题日益突出。单纯依靠软件的安全机制现在已经不能够充分保护信息的安全，可信计算组织提出为计算平台增加具有安全保护功能的硬件，基于此硬件通过软硬件结合的方式构建可信计算环境，构建的可信环境可安全有效得实现密钥封装、内存屏蔽、封装存储、远程证明等安全技术，更好解决计算机安全问题。

可信计算中的具有安全保护功能的硬件就是可信平台模块TPM（Trusted Platform Moudle），由可信计算组织TCC（Trusted Computing Group）指定详细规范，由规定的模块组成，能实现加密、解密、认证、保证私钥安全、检测恶意代码等安全功能，在硬件层执行保护，能保证软件层的非法访问和恶意操作，是构建可信环境的出发点。

可信计算平台基于可信平台模块TPM（Trusted Platform Modules），以密码算法技术作为基础、安全操作系统作为核心，通过信任域不断扩展形成的安全平台。可信平台组成如图 11


图 11可信平台框架

Linux 内核从2.6.12开始加入TPM芯片驱动程序，可使Linux系统使用TPM芯片来构建可信平台，增强Linux系统的安全性。Linux Kernel中的TPM设备驱动是字符设备驱动。本文分析了TPM设备的模块组成及相应功能，分析了Linux3.5.4中的字符设备驱动程序框架及主要函数实现，分析了TPM设备驱动程序各模块的关系及TPM驱动程序中主要函数的实现过程和调用层次。

1.1 涉及到的源码范围说明

在linux 内核中，TPM驱动程序是字符设备驱动程序。在linux 3.5.4中，与字符设备驱动有关的文件有：/fs/char_cev.c、/include/linux/fs.h、/include/linux/cdev.h等，TPM驱动程序位于/drivers/char/tpm目录下。文件详细介绍如下。

/include/linux/fs.h：文件操作的相关结构体和函数的定义

/include/linux/cdev.h：字符设备驱动中相关结构体和操作函数的定义。

/fs/char_dev.c：字符设备驱动中的主要文件，包含操作字符设备的主要函数的源码实现。

/driver/char/tpm/tpm.h：TPM设备驱动程序中，与TPM设备通信的标准统一接口及相关数据结构的定义。

/driver/char/tpm/tpm.c：与TPM设备通信的标准统一接口的源码实现。

/driver/char/tpm_atmel.c：支持TPM1.1b规范的atmel公司TPM芯片操作函数的源码实现。

/driver/char/tpm_infineon.c：支持TPM1.1b规范的infineon公司TPM芯片操作函数的源码实现。

/driver/char/tpm_nsc.c：支持TPM1.1b规范的nsc公司TPM芯片操作函数的源码实现。

/driver/char/tpm_tis.c：符合TPM1.2规范的TPM设备的操作函数的源码实现。

1.2 技术方案及原理

文档从TPM设备出发，分析了TPM设备模块组成及相应的功能，说明TPM设备是构建可信计算平台的可靠的基础。TPM设备驱动程序在Linux内核中是字符设备驱动，文中分析了字符设备驱动的构建和主要函数的实现流程。分析了Linux内核中TPM设备驱动程序的模块构架和实现原理、主要函数的实现过程。设计的技术原理如下。

1.可信平台模块TPM

可信计算是一个很广泛的范畴，可信平台从硬件到操作系统再到上层应用软件。可信计算的出发点是可信计算平台模块TPM，可信计算技术的实现都要基于可信平台模块TPM。TPM功能模块组成按TCG规范定义，框架如图 12。


 

图 12  TPM功能模块框架图

(1) Non-Volatile Storage

非易失性存储(Non-Volatile Storage)用于保存在断电情况下仍需保存在TPM内部的数据。这其中主要包括Endorsement Key(EK)和Storage Root Key(SRK)。EK是在TPM芯片出厂时预置的RSA密钥，作为TPM的唯一性密码学身份标识，唯一地标识了一个TPM的身份，一旦生成就会固化到安全芯片上，不允许再修改。SRK也是一对RSA密钥，用于加密保存在TPM外部的密钥，与EK不同的是它可以改变。

(2) Attestation Identity Keys

身份密钥(Attestation Identity Keys)是永久的，其存储在TPM外部的永久存储器中，而不是存储在TPM的非易失存储器中。

(3) Program Code

程序代码（Pragram Code）含有测量平台设备的固件。

(4) Random Number Generator

随机数产生器(Random Number Generator) 用于产生对称密钥、非对称密钥或随机数等，以加强密码系统的强度。

(5) Sha-1 Engine

Sha-1算法引擎(Sha-1 Engine) 是一个Sha-1消息摘要引擎，用于计算数字签名值、建立密钥等。

(6) Key Generation

密钥产生器(Key Generation)用于产生数字签名密钥与资料存储密钥等密钥。

(7) RSA Engine

RSA引擎(RSA Engine)用签名密钥签名、用存储密钥加密/解密等。

(8) Opt-In

Opt-In选择组件主要是存储TCG策略、TPM模块运行模式，以符合消费者的需求。

(9) Execution Engine

执行引擎(Execution Engine)运行程序代码，执行TPM初始化和测量操作。

(10) Power Detection

功率检测 (Power Detection)将所有关于能量状态变化的信息通知给TPM。

(11) I/O

I/O组件管理通信总线上的信息流，它执行协议的编码/解码，并将消息送给TPM合适的组件。

TPM设备能够保证私钥安全、检测恶意代码、保证加密密钥安全，具有安全度量及报告、远程证明、数据保护、密钥管理等功能，在硬件级别执行信息保护，是构建可信计算环境可靠的基础，基于TPM设备使用软硬件结合的方式可以构建可靠有效的可信计算平台，实现封装存储、内存屏蔽、安全输入输出、远程证明等安全技术。平台中最核心的信息受到TPM设备硬件级别的保护，除非将TPM设备硬件从平台上移除，否则很难突破硬件层的保护，这样大大增加了系统平台的安全性。
