对于该文件定义的相关函数，下面进行详细解释：

①　selinux_set_mapping()

函数原型：static int selinux_set_mapping(struct policydb *pol,

   struct security_class_mapping *map,

                       struct selinux_mapping **out_map_p,

   u16 *out_map_size)；

函数参数：pol表示SELinux所使用的策略库；map表示客体类别与权限的映射表；out_map_p用于存放策略中的类别与权限的映射关系；out_map_size表示out_map_p的长度。

函数功能：计算参数map中客体类别的数量，并将map中字符形式的类别-权限映射转换为数值形式的类别-权限映射。

返回值：成功时返回0；反之返回相应的错误码。

②　unmap_class()

函数原型：static u16 unmap_class(u16 tclass)；

函数参数：tclass表示内核中的客体类别值。

函数功能：将内核中的客体类别值转换为策略中相应的类别值。

返回值：返回策略库中类别值。

③　map_class()

函数原型：static u16 map_class(u16 pol_value)；

函数参数：pol_value表示策略中的客体类别值。

函数功能：将客体类别在策略中的值转换为内核中相应的类别值。

返回值：返回内核中相应的类别值。

④　map_decision()

函数原型：static void map_decision(u16 tclass, struct av_decision *avd,

   int allow_unknown)；

函数参数：tclass表示客体类别值；avd用于存放访问向量决策；allow_unknown表示不能识别的权限。

函数功能：根据客体类别获取相应的访问向量决策。

返回值：无。

⑤　security_mls_enabled()

函数原型：int security_mls_enabled(void)；

函数参数：无。

函数功能：判断是否启用了MLS策略。

返回值：如果启用了MLS，则返回真；反之返回假。

⑥　constraint_expr_eval()

函数原型：static int constraint_expr_eval(struct context *scontext,struct context *tcontext,

                           struct context *xcontext,

   struct constraint_expr *cexpr)；

函数参数：scontext表示主体的安全上下文；tcontext表示客体的安全上下文；xcontext只用于validatetrans规则，用来表示执行转换的进程的安全上下文；cexpr表示待计算的约束表达式。

函数功能：计算约束表达式的布尔值。

返回值：返回表达式的值。

⑦　security_dump_masked_av()

函数原型：static void security_dump_masked_av(struct context *scontext,

  struct context *tcontext,u16 tclass,

  u32 permissions,const char *reason)；

函数参数：scontext表示主体的安全上下文；tcontext表示客体的安全上下文；tclass表示客体的类别；permissions表示待审计的权限；reason表示相应的错误的原因。

函数功能：审计指定的权限。

返回值：无。

⑧　context_struct_compute_av()

函数原型：static void context_struct_compute_av(struct context *scontext,

  struct context *tcontext,u16 tclass,

  struct av_decision *avd);

函数参数：scontext表示源安全上下文；tcontext表示目标安全上下文；tclass表示客体类别；avd表示访问向量决策。

函数功能：根据客体类别和安全上下文对计算访问向量。

返回值：无。

⑨　security_validate_transition()

函数原型：int security_validate_transition(u32 oldsid, u32 newsid, u32 tasksid,

u16 orig_tclass)；

函数参数：oldsid表示之前的安全标识符；newsid表示新指定的安全标识符；tasksid表示进程对应的安全标识符；orig_tclass表示客体类别。

函数功能：判断指定的源、目的、进程的安全上下文是否满足客体类别的约束表达式，若不满足，则释放相应的安全上下文对应的空间。

返回值：成功时返回0；失败时返回相应的错误码。

⑩　security_bounded_transition()

函数原型：int security_bounded_transition(u32 old_sid, u32 new_sid)；

函数参数：oldsid表示当前的安全标识符；newsid表示新指定的安全标识符。

函数功能：检查指定的转换是否受到限制。

返回值：如果newsid受到oldsid的约束，返回0，否则返回错误码。

11　avd_init()

函数原型：static void avd_init(struct av_decision *avd)；

函数参数：avd表示待处理的访问向量决策。

函数功能：初始化访问向量决策。

返回值：无。

12　security_compute_av()

函数原型：void security_compute_av(u32 ssid,u32 tsid,u16 orig_tclass,

struct av_decision *avd)；

函数参数：ssid表示主体的SID；tsid表示客体的SID；orig_tclass表示客体的类别；avd用于存放相应的访问向量决策。

函数功能：根据客体类别对(ssid,tsid)计算访问向量决策。

返回值：无。

13　context_struct_to_string()

函数原型：static int context_struct_to_string(struct context *context, char **scontext,

  u32 *scontext_len)；

函数参数：context表示待处理的安全上下文；scontext指向存放字符串形式的安全上下文的缓冲区；scontext_len表示字符串形式的安全上下文的字节长度。

函数功能：将一个安全上下文字符串写入到动态分配的字符串中，并让参数*scontext指向该字符串，*scontext_len存放该字符串的长度。

返回值：成功时返回0；错误时返回相应的错误码。

14　security_get_initial_sid_context()

函数原型：const char *security_get_initial_sid_context(u32 sid)；

函数参数：sid表示待处理的初始SID。

函数功能：将初始化SID转化为相应的上下文字符串。

返回值：返回初始SID对应的安全上下文。

15　security_sid_to_context_core()

函数原型：static int security_sid_to_context_core(u32 sid, char **scontext,

  u32 *scontext_len,int force)；

函数参数：sid表示待处理的安全标识符；scontext指向存放字符串形式的安全上下文的缓冲区；scontext_len表示字符串形式的安全上下文的字节长度；force表示一个标志。

函数功能：将SID关联的安全上下文字符串写入动态分配的字符串中，并让scontext指向该字符串，scontext_len存放该字符串的长度。

返回值：成功时返回0；失败时返回相应的错误码。
