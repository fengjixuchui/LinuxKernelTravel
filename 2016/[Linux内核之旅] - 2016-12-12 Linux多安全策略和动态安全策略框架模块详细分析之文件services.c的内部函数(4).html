16　string_to_context_struct()

函数原型：static int string_to_context_struct(struct policydb *pol,

  struct sidtab *sidtabp,char *scontext,

                                  u32 scontext_len,struct context *ctx,

  u32 def_sid)；

函数参数：pol表示SELinux所使用的策略库；sidtabp表示策略库中的SID映射表；scontext表示字符串形式的安全上下文；scontext_len表示字符串形式的安全上下文的长度；ctx用于存放字符形式的安全上下文转换成的结构体形式的安全上下文；def_sid表示默认的SID。

函数功能：解析安全上下文字符串，将其转化为相应的context结构。

返回值：成功时返回0；失败时返回相应的错误码。

17　security_context_to_sid_core()

函数原型：static int security_context_to_sid_core(const char *scontext,

  u32 scontext_len,u32 *sid,

  u32 def_sid, gfp_t gfp_flags,int force)；

函数参数：scontext表示字符串形式的安全上下文；scontext_len表示字符串形式的安全上下文的字节长度；sid表示安全上下文对应的安全标识符；def_sid默认的SID；gfp_flags表示内存分配的标志；force表示一个标志。

函数功能：获取字符串形式的安全上下文对应的SID，成功时返回0。

返回值：成功时返回0；失败时返回相应的错误码。

函数功能：对于一个无效的安全上下文生成一条审计记录。

19　filename_compute_type()

    const char *objname)；

20　security_compute_sid()

函数原型：static int security_compute_sid(u32 ssid,u32 tsid,u16 orig_tclass,u32 specified,

            const char *objname,u32 *out_sid,bool kern)；

函数参数：ssid表示主体的SID；tsid表示客体的SID；orig_tclass客体的安全类别；specified指定了访问向量的类型；objname表示对象的名字；out_sid用来存放新建的对象的安全标识符；kern标识表示相应的客体类别值是否是内核中的值。

函数功能：根据源SID、目标SID以及目标客体类别生成一个安全上下文，并获取该上下文对应的SID。

返回值：成功时返回0；失败时返回相应的错误码。

函数功能：将SID和其对应的安全上下文插入到SID表中。

返回值：成功时返回0；失败时返回相应的错误码。

22　convert_context()

函数原型：static int convert_context(u32 key,struct context *c,void *p)；

函数参数：key表示待修改的安全上下文对应的SID；context表示待修改的安全上下文；p表示策略变化前后所使用的策略。

函数功能：将安全上下文c中的值由p->oldp指定的值转化为p->newp指定的值，并基于新策略确定安全上下文的有效性。

返回值：成功时返回0；失败时返回相应的错误码。
