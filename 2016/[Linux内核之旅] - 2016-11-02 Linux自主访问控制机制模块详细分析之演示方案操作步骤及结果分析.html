3.1.4 操作步骤

3.1.4.1 未运行演示程序时文件的创建

1. 其他用户对待测试的目录没有写权限时文件的创建

(1) 进入待测试的目录，即test目录。

(2) 切换到cxp用户。

(3) 在该目录中创建一个名为cxp1的文件。

2. 其他用户对待测试目录有写权限时文件的创建

(1) 查看其他用户对test的执行权限，如果没有写权限，则授予写权限。

(2) 切换到cxp用户。

(3) 进入待测试的目录，即test目录。

(4) 在该目录中创建一个名为cxp1的文件。


3.1.4.2 运行演示程序之后文件的创建

(1) 在一个终端中运行演示程序。

(2) 启动另一终端，在该终端中查看cxp对test目录的操作权限，如果没有写权限，则授予其写权限。

(3) 切换到cxp用户。

(4) 在test目录中创建名为cxp2的文件。


3.1.5 实验代码

#include <stdio.h>

#include <sys/inotify.h>

#include <string.h>

 

#define MONITORED_PATH "."

#define MONITORED_EVENT IN_ATTRIB

#define BUFFER_SIZE 256

#define COMMAND_SIZE 256

 

int main(int argc, char *argv[])

{

 int fd,wd,len;

 char buf[BUFFER_SIZE],command[COMMAND_SIZE];

 char *monitored_path = MONITORED_PATH;

 struct inotify_event *event;


 fd = inotify_init();

 if (-1 == fd) {

   perror("inotify_init error!");

   return -1;

 }

 if (argc > 1)

   monitored_path = argv[1];

 memset(command, 0, sizeof(command));

 strcpy(command, "setfacl -m u:cxp:rx ");

 strcat(command, monitored_path);

 system(command);

 wd = inotify_add_watch(fd, monitored_path, MONITORED_EVENT);

 if (-1 == wd) {

   perror("inotify_add_watch error!");

   return -1;

 }

 memset(buf, 0, sizeof(buf));

 while (len = read(fd, buf, BUFFER_SIZE)) {

   event = (struct inotify_event *)buf;


   while ((char *)event - buf < len) {

     if (event->mask & MONITORED_EVENT) {

       printf("change the permission!\n");

       system(command);

  event = (struct inotify_event *)((char *)event + sizeof(struct inotify_event) + event->len);

       continue;

     }

  event = (struct inotify_event *)((char *)event + sizeof(struct inotify_event) + event->len);


   }

   memset(buf, 0, sizeof(buf));

 

 }

 return 0;

}

由上可知，在执行了演示程序之后，即使使用chmod命令授予了其他用户写权限，并且通过ls命令显式的看见了写权限，当使用su命令切换到cxp时，还是不能创建文件，并且在执行chmod命令后，会发现执行演示程序的终端显式“change the permission！”。这是因为在演示程序中使用了inotify机制对文件系统进行监视，当用户调用chmod命令修改test的执行权限时，inotify机制便会捕捉到该行为，并生成一条警告信息来通知用户程序，用户程序发现该警告之后，便调用setfacl命令删除cxp用户对test目录的写权限。又因为文件权限的检查是先进行普通权限的检查，然后进行自主访问控制检查，最后进行强制访问控制检查，所以即使使用chmod授予权限后显示的看见了写权限，但其并未通过自主访问控制检查，所以文件的创建还是以失败告终。
