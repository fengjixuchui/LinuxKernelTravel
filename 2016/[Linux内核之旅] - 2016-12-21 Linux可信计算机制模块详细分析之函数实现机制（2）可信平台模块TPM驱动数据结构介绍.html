在linux-3.5.4中，用结构 tpm_chip来描述TPM芯片，在/drivers/char/tpm/tpm.h文件中定义，如下所示。

struct tpm_chip {

struct device *dev;/* Device stuff */

int dev_num;/* /dev/tpm# */

unsigned long is_open;/* only one allowed */

int time_expired;

/* Data passed to and from the tpm via the read/write calls */

u8 *data_buffer;

atomic_t data_pending;

struct mutex buffer_mutex;

struct timer_list user_read_timer;/* user needs to claim result */

struct work_struct work;

struct mutex tpm_mutex;/* tpm is processing */

struct tpm_vendor_specific vendor;

struct dentry **bios_dir;

struct list_head list;

void (*release) (struct device *);

};

表2-5  tpm_vendor_specific结构体各字段说明

Linux 支持多个TPM 设备，每个结构tpm_chip实例代表一个TPM设备，系统中第一个TPM设备的位置是/dev/tpm0，第二个位置是/dev/tpm1，以此类推。Linux使用全局链表tpm_chip_list 存放多个tpm_chip实例。

TPM设备驱动程序的探测函数（probe）调用tpm_register_hardware()函数将设备注册到链表tpm_chip_list中，tpm_register_hardware()函数会给tpm_chip实例分配空间，将结构初始化，设置工作函数等。tpm_remove_hardware()函数从链表tpm_chip_list中删除TPM设备实例，并释放设备结构所在空间。
