2.4.1.4 外部函数

在statfs.c文件中主要涉及到的外部函数有：

1. 函数名称：memcpy（）

函数原型：void * memcpy(void * dest, const void *src, size_t n)

函数功能：从所指的内存地址的起始位置开始拷贝n个字节到目标所指的内存地址的起始位置中。

参数：dest表示要拷贝去的目标所指向的内存地址开始位置。

      src表示要拷贝的内存的起始位置。

      n表示要拷贝的字节数。

返回值：返回dest的值。

在statfs.c文件中函数do_statfs_native（）,do_statfs64（）都是通过调用该函数来讲磁盘的相关信息拷贝到用户空间去的。

2. 函数名称：memset（）

函数原型：void *memset(void *s, int c, size_t count)

函数功能：在一段内存块中填充给定的值，它是对较大的结构体或数组进行清零操作的一种最快方法。

参数：s表示要被替换掉的目标。

      n表示要被替换掉的目标的前n个字节。

      count表示进行计数。

返回值：无返回值。

3. 函数名称：fget_light（）

函数原型：struct file *fget_light(unsigned int fd, int *fput_needed)

函数功能：函数通过文件描述符返回对应的文件结构。

参数：fd表示指定的文件描述符

      fput_needed表示指向需要的文件结构。

返回值：无返回值

4. 函数名称：copy_to_user（）

函数原型：unsigned long copy_to_user(void __user *dst, const void *src, unsigned long len)

函数功能：从指定的地址开始位置拷贝n个字节到目标所指用户态地址位置中。

参数：dst表示要拷贝的去的目标所指向的地址。

      src表示要拷贝内容的起始地址位置。

      len表示要进行拷贝的长度。

返回值：返回dst的值。

5. 函数名称：drop_super（）

函数原型：void drop_super(struct super_block *sb)

函数功能：删除一个临时参考超级块。

参数：sb表示要删除的临时参考超级块。

返回值：无返回值

2.4.1.5 内部函数

1. 函数名称：statfs_by_dentry（）

函数原型：static int statfs_by_dentry(struct dentry *dentry, struct kstatfs *buf)

函数功能：获取目录项的相关信息。

参数：dentry表示要获取相关信息的目标目录。

      buf表示指向存储相关信息结构体的指针。

返回值：执行成功返回0，失败返回-1.

函数的调用过程如图所示:


 

图2-10：函数statfs_by_dentry（）流程图

对应流程图来说明函数的执行步骤如下：

(1). 该函数首先判断对目录的操作指针是否指向statfs，如果不是则返回一个错误即功能未实现，如果是则进行下一步。

(2). 将kstatfs结构体中的信息进行清零操作，清零工作通过调用函数memset（）来完成。

(3). 接下来对通过函数security_sb_statfs（）该函数通过返回值将安全操作指向sb_statfs，然后再将操作指向statfs得到关于目录的相关信息。

1. 函数名：vfs_statfs（）

函数原型：int vfs_statfs(struct path *path, struct kstatfs *buf)

函数功能：获得虚拟文件系统的相关信息。

参数：path表示需要得到虚拟文件系统相关信息的路径。

      buf表示指向存储相关信息结构体的指针。

返回值：返回dentry。

函数说明：该函数通过调用函数statfs_by_dentry获取相关信息，然后对标志位进行判断。

函数的调用过程如图所示:


 

图2-11：函数vfs_statfs（）流程图

函数说明：该函数通过调用函数statfs_by_dentry获取相关信息，然后对标志位进行判断。

1. 函数名称：do_statfs_native（）

函数原型：static int do_statfs_native(struct kstatfs *st, struct statfs __user *p)

函数功能：获取磁盘的相关信息，并拷贝到用户态。

参数：st表示指向存储相关信息结构体的指针。

      p表示指向要拷贝给的用户。

返回值：执行正确返回0，执行错误返回-1。

函数的调用过程如图所示:


 

图2-12：函数do_statfs_native（）流程图

对应流程图来说明函数do_statfs_native（）执行过程：

(1). 首先判断kstatfs结构体和statfs结构体的字节数是否相等，如果相等直接调用函数memcpy（）将kstatfs中的字节拷贝到statfs结构体中，如果不相等则进行一系列的溢出判断，该函数只针对32位系统所以如果字段溢出32位就出错。否则分别将结构体kstatfs中的值付给statfs结构体。

(2). 通过函数copy_to_user（）将statfs结构体中的到得磁盘相关信息拷贝到用户态，拷贝成功返回0，拷贝失败返回错误：错误的地址。

2. 函数名称：fd_statfs（）

函数原型：int fd_statfs(int fd, struct kstatfs *st)

函数功能：获取文件在虚拟文件系统中的信息。

参数：fd所要获取文件信息的文件对应的fd。

      st指向kstatfs结构体的指针。

返回值：执行成功返回0，失败返回-1。

函数流程图：


 

图2-13:函数fd_statfs（）流程图

对应流程图来说明函数fd_statfs执行过程

(1). 通过函数fget_light（）获取参数fd对应的file结构。

(2). 确认是否成功获取file结构，如果错误返回错误的文件数，如果正确继续执行下一步。

(3). 通过函数vfs_statfs（）获取文件在虚拟文件系统的相关信息。

(4). 调用函数fput_light（）释放file结构。

3. 函数名称：calculate_f_flags（）

函数原型：static int calculate_f_flags(struct vfsmount *mnt)

函数功能：计算文件系统的标志位信息。

参数：mnt指向已挂载的文件系统。

返回值：返回文件系统的标志位。

4. 函数名称：do_statfs64（）

函数原型：static int do_statfs64(struct kstatfs *st, struct statfs64 __user *p)

函数功能：获取磁盘的相关信息，并拷贝到用户态，供64位系统使用。

参数：st表示指向kstatfs结构体的指针。

      p表示指向要输出的用户。

函数说明：该函数的流程与do_statfs_native（）函数相似，因为是供64位系统使用的所以少了溢出的判断。
