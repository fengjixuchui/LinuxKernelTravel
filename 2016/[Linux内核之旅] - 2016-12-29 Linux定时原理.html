Linux可以通过alarm()或者setitimer()系统调用创建一个定时器，有PHP编程经验的朋友应该都知道PHP可以设置脚本执行的超时时间，而这个功能就是使用Linux的setitimer()来实现的。


一、定时器的使用

我们简单了解下PHP的超时功能是怎样实现的，从而对Linux的定时器有个初步的认识：

void zend_set_timeout(long seconds)

{

    ...

    struct itimerval t_r;

    sigset_t sigset;

 

    t_r.it_value.tv_sec = seconds;

    t_r.it_value.tv_usec = 0;

t_r.it_interval.tv_sec = 0;

t_r.it_interval.tv_usec = 0;

 

    setitimer(ITIMER_PROF, &t_r, NULL);

    signal(SIGPROF, zend_timeout);

    ...

}

去除一些与功能无关的细节，我们可以看到PHP就是通过setitimer()系统调用创建一个定时器，而定时器的计时时间就是从zend_set_timeout()函数参数传递过来的seconds。经过seconds秒后，系统就会发送一个SIGPROF的信号给当前进程。因为在前面已经通过signal()系统调用设置了SIGPROF信号的回调函数为zend_timeout()，所以这时就会调用zend_timeout()函数。

 

我们再来看看zend_timeout()函数的代码：

ZEND_API void zend_timeout(int dummy)

{

    ...

    zend_error(E_ERROR, "Maximum execution time of %d second%s exceeded",

              EG(timeout_seconds), EG(timeout_seconds) == 1 ? "" : "s");

}

zend_timeout()的代码很简单，就是打印一句PHP程序员都可能遇过的 "Maximum execution time of xxx seconds exceeded"，然后退出当前的执行。


二、Linux定时器的实现

知道了定时器怎么时候后，再来看看定时器在内核是怎么实现的。

当我们在程序调用setitimer()系统调用设置一个定时器时，会陷入到内核态，并调用内核态的sys_setitimer()函数。sys_setitimer()函数代码如下：

asmlinkage long sys_setitimer(int which, struct itimerval *value,

                  struct itimerval *ovalue)

{

    ...

    error = do_setitimer(which, &set_buffer, ovalue ? &get_buffer : 0);

    if (error || !ovalue)

        return error;

    ...

    return 0;

}

sys_setitimer()函数的核心部分就是调用do_setitimer()函数设置一个定时器。do_setitimer()函数的代码如下：

int do_setitimer(int which, struct itimerval *value, struct itimerval *ovalue)

{

    register unsigned long i, j;

    int k;

 

    i = tvtojiffies(&value->it_interval);

    j = tvtojiffies(&value->it_value);

    ...

    switch (which) {

        ...

        case ITIMER_PROF:

            if (j)

                j++;

            current->it_prof_value = j;

            current->it_prof_incr = i;

            break;

        ...

    }

    return 0;

}

    我们只留下相关的代码，可以看出do_setitimer()只是设置当前进程（current）的it_prof_value和it_prof_incr字段，而并没有其他触发定时器的逻辑，那么触发定时器的逻辑在哪里呢？

在Linux内核中，一般与时间相关的都需要时钟中断来支持的，所以不难发现，定时器也是由时钟中断触发的。在上一篇文章《Linux延时实现》说过，时钟中断是由硬件提供的一个功能，其功能就是每隔一个时间段发生中断，并调用中断处理函数。时钟中断的处理函数是do_timer()，代码如下：

void do_timer(struct pt_regs *regs)

{

    (*(unsigned long *)&jiffies)++;

 

    update_process_times(user_mode(regs));

 

    ...

}

在上面代码中，与定时器相关的就是update_process_times()函数，而update_process_times()函数会调用update_one_process()函数，而update_one_process()最终会调用do_it_prof()。

do_it_prof()函数代码如下：

static inline void do_it_prof(struct task_struct *p)

{

    unsigned long it_prof = p->it_prof_value;

 

    if (it_prof) {

        if (--it_prof == 0) {

            it_prof = p->it_prof_incr;

            send_sig(SIGPROF, p, 1);

        }

        p->it_prof_value = it_prof;

    }

}

do_it_prof()函数首先会把进程it_prof_value减一，然后看看是否为0，如果为0代表已经超时，并调用send_sig()函数发送一个SIGPROF信号给进程，此时进程便接收到SIGPROF信号，并会调用信号相关的处理函数。
