3. 重要函数的分析

如上文所介绍的，为了提高对象管理器的性能，Flask体系结构在对象管理器中引入了访问向量缓存，上一小节已经对其相关操作的功能进行了简单的描述，这里主要针对其中比较重要的函数进行详细的分析：

(1)  avc_lookup()

avc_lookup()函数用于基于(ssid,tsid)查找访问向量，若存在，返回包含该访问向量的结点，反之返回NULL。其函数头如下所示：

static struct avc_node *avc_lookup(u32 ssid, u32 tsid, u16 tclass)

该函数有3个参数：ssid表示主体的SID；tsid表示客体的SID；tclass表示客体的类别。对于该函数，其主要调用了avc_search_node()函数，后者用于根据源SID、目标SID、目标的类别在访问向量缓存中查找指定的结点，如果找到，则返回相应的结点；否则返回NULL。对于avc_search_node()，其函数头如下所示：

static inline struct avc_node *avc_search_node(u32 ssid, u32 tsid, u16 tclass)

该函数包含3个参数，其含义和avc_lookup()函数系统，这里不再赘述。下面结合源码对其主要执行步骤进行说明：


①　调用avc_hash()函数获取参数指定的访问向量在哈希表中的哈希值。

②　调用hlist_for_each_entry_rcu()宏来遍历具有相同哈希值的访问向量组成的链表，如果某个访问向量中指定的源SID、目标SID、目标类别均与待查找的访问向量相同，则该访问向量即为要查找的访问向量，然后返回该节点。对于hlist_for_each_entry_rcu()宏，其实际上是一个for循环。

③　结束并返回。

 

(2)  avc_insert()

avc_insert()函数用于根据指定的源SID、目标SID、客体类别向访问向量缓存中插入一个访问向量，成功执行时返回插入的访问向量，反之返回NULL。函数头如下所示：

static struct avc_node *avc_insert(u32 ssid, u32 tsid, u16 tclass, struct av_decision *avd)

该函数包含4个参数：ssid表示源SID；tsid表示目标SID；tclass表示客体类别；avd表示相应的访问决策。对于该函数，其函数调用流程图如下图2-17所示，下面结合源码对该函数主要执行步骤进行说明：


①　判断avd中的序列号是否大于访问向量缓存中的latest_notif，若是，直接返回NULL。

②　调用avc_alloc_node()函数分配一个新的AVC节点，如果该函数成功执行，即成功的分配了一个AVC结点,则按如下步骤进行处理：

a. 调用avc_hash()函数根据参数计算待插入的访问向量在哈希表中的存放位置。

b. 调用avc_node_populate()函数来根据参数填充指定的AVC结点，即刚分配的结点。

c. 调用hlist_for_each_entry()宏在访问向量缓存中查找参数指定的访问向量，如果存在，则调用avc_node_replace()函数用node结点替换该节点，然后返回node。

d. 调用hlist_add_head_rcu()函数将node节点插入到访问向量缓冲中。

③　结束并返回node。


图2-17 avc_insert()函数调用流程图

(3)  avc_ss_reset()

avc_ss_reset()函数用于更新访问向量缓存，函数头如下所示：

int avc_ss_reset(u32 seqno)

该函数只有1个参数：seqno表示策略序列号。对于该函数，下面结合源码对其主要执行步骤进行说明：


①　调用avc_flush()函数刷新访问向量缓存。

②　遍历访问向量缓存注册的事件回调函数列表，从中查找AVC_CALLBACK_RESET事件对应的回调函数，如果找到，则执行相应的回调函数。

③　调用avc_latest_notif_update()函数更新访问向量缓存中的撤销通知序列号latest_notif。

④　结束并返回。

 

(4)  avc_has_perm_noaudit()

avc_has_perm_noaudit()函数用于在不执行审计的情况下检查访问向量缓存来确定是否授予SID对(ssid,tsid)以请求的权限，如果在AVC中没有找到指定的决策，则查询安全服务器，并将相应的决策插入到AVC中；如果授予了指定的权限，则返回0。对于该函数，其函数头如下所示：

inline int avc_has_perm_noaudit(u32 ssid, u32 tsid,u16 tclass, u32 requested,

                          unsigned flags,struct av_decision *avd)


该函数有6个参数：ssid表示源SID；tsid表示目标SID；tclass表示目标类别；requested表示请求的权限；flags表示一个标志，只能是AVC_STRICT或0；avd表示访问向量决策。


图2-18 avc_has_perm_noaudit()函数调用流程图

如图2-18所示，下面结合对该函数的主要执行步骤进行说明：

①　调用avc_lookup()函数在访问向量缓存中查找指定的SID对对应的访问向量结点，并将查找结果存放到node中。

②　判断node是否为空，如果为空，则调用avc_compute_av()函数来在安全服务器查找对应的访问向量，并将找到的访问向量插入到访问向量缓存中；否则将找到的访问向量结点中的访问向量决策拷贝到参数avd中。

③　判断是否请求了访问向量未授予的权限，若是，则调用avc_denied()函数将相应的权限授予SID对。

④　结束并返回。
