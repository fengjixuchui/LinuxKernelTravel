2.3.2 对象管理器

对象管理器主要由三部分组成：决策检索接口、访问向量缓存和策略变化通知接口，其中决策检索接口和策略变化通知接口均由访问向量缓存来进行维护，因此下文首先对访问向量缓存进行介绍，随后以文件系统为例对对象管理器的实施进行介绍。


2.3.2.1 AVC

1. AVC的数据结构

在SELinux中，访问向量缓存用avc_cache数据结构表示，其定义在security/selinux.avc.c中，具体定义如下：

struct avc_cache {

    struct hlist_head     slots[AVC_CACHE_SLOTS]; /* head for avc_node->list */

         spinlock_t           slots_lock[AVC_CACHE_SLOTS]; /* lock for writes */

         atomic_t            lru_hint;       /* LRU hint for reclaim scan */

         atomic_t            active_nodes;

         u32                latest_notif;   /* latest revocation notification */

  };


如上所述，代码中已对各字段的含义进行了解释，这里不再赘述。对于访问向量列表，其实际上是由访问向量缓存节点组成的哈希表，每个节点使用定义在security/selinux.avc.c中的avc_node结构表示，具体定义如下：

struct avc_node {

           struct avc_entry        ae;

           struct hlist_node       list; /* anchored in avc_cache->slots[i] */

       struct rcu_head         rhead;

   };

对于avc_entry结构，其代表一个具体的访问向量，该结构定义在security/selinux.avc.c中，具体定义如下：

struct avc_entry {

          u32                     ssid;

          u32                     tsid;

          u16                     tclass;

          struct av_decision         avd;

   };

其中av_decision结构用来表示访问向量决策，该结构定义了相应主体对指定客体的的访问权限，如允许的权限、允许但需要审计的权限、拒绝但须审计的权限等等，其定义在security/selinux/ss/security.h中，具体定义如下：

struct av_decision {

          u32 allowed;

          u32 auditallow;

         u32 auditdeny;

         u32 seqno;

         u32 flags;

  };
