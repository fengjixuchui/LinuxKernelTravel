上一讲中给大家介绍了Java虚拟机JVM，有了Java虚拟机这个规范之后，出现了一个有趣的局面，换句话说，我们可以用非Java语言来编写Java程序。也就是把其他语言写出的代码通过编译技术将代码转换为.Class文件，也就是Java虚拟机能够接受的文件格式，这样用其他语言编写的程序便可以在Java虚拟机上运行，因此开发出了许多基于Java虚拟机的语言。在这里我列举几个语言，例如：第一个是Clojure(Lisp现代的一个版本)；第二个是Scala,由瑞士的威尔逊教授的博士生马丁.奥德斯基（Martin Odersky）来开发的；第三个是Python，用Java的虚拟机写了一个Python的Java实现，称为Jython；第四个是javascript，它在发明阶段，和Java并未有任何关系，只是在名称上有所不同，目前为止，在Java虚拟机上已经实现了Javascript ，而且在Oracle 公司发布的JDK中也有Javascript 的实现，叫做JJS；第五个是Groovy，Groovy被用于Java应用程序的构建系统去替代像写Makefile这样的工具；第六个是Ruby，它是由日本人松本行弘(Yukihiro Matsumoto)所开发的，在Ruby流行一段时间以后，Ruby社团也用Java虚拟机来实现Ruby，被称作JRuby。当然还有许多语言也是工作在Java虚拟机上。在这里就不去一一列举了。

美国一家计算机图书出版商曼宁,在2013年的时候出版一本书叫做《The well-grounded java developer》，在这本书的第三个部分提到了一个有趣的概念说Polyglot programming on JVM，Polyglot是一个能够讲多种语言的虚拟机。在Java虚拟机上诞生了如此之多的语言之后，出现了新的一种编程形式就是基于Java虚拟机的多种语言编程，现在我可以把我的关于用哪种语言来开发应用程序的观点和盘托出。前面我粗略的讲过，用java语言来设计大规模的应用程序，现在有了Java语言和Java虚拟机之后，我们可以围绕Java虚拟机去选择最合适的语言去开发应用程序。下面我向大家介绍编程范式下的一些语言，包括Clojure，Scala,Groovy，Javascript，关于Jsthon会在讲Python语言的时候回到这个主题。

我们首先来介绍下Clojure，关于Clojure我们可以从5个方面来看待。第一个点是Lisp的一个实现，所有Lisp的优势在我前面所讲的Racket，Commom Lisp，Emacs Lisp在Clojure中能够找到其对应的特点。第二点Clojure支持函数式的编程，函数式编程实际上在Lisp早期就已经有了，Clojure将函数式的编程抬到一个高度,用Clojure去开发java应用的时候会非常的安全，发挥了函数式编程的好处。第三点Clojure工作在成熟的，有效率的运行式java虚拟机上，这也是其迅速流行的原因。与此相关的第四点是Java和Clojure的互操作非常容易，在Clojure中调用Java当中的类库，反过来在Java程序中调用Clojure中的代码，双方的互操作是极为流畅的。最后一点是Clojure为并发和平行计算提供了很好的支持。在这里比较一下Java和Clojure的开发环境，在Clojure里面有REPL（Read Evaluate print loop）即读取用户的输入程序，求值，打印，再一次循环等待用户的下一次操作。这种开发模式对程序员提高生产力是十分有益的，相比Java的编程模式先用编辑器编写源代码，再用Javac编译器来生成.Class文件，然后运行，运行过程出现问题了再回过头来进行修改，相比Clojure工作模式慢多了。原来的Lisp程序员为什么会傲视其他程序员呢？是因为他们已经在REPL这个环境下工作了许久。Pual Graham这个程序员是Viaweb创业项目的合作人之一，后来在黑客与画家《Hackers and Painters》这本书中写道，写程序的过程和画家作画的过程极为相似，画家在作画的过程中先要打草稿，反复修改，最终得到一个作品。软件开发人员在开发软件的时候也不是一次到位的，也存在反复修改，调试，改进的过程。与画家作画的过程极为相似。在这种模式下，通过解释的方式去开发程序是非常快的，因为程序的对错计算机会马上告诉结果，然后根据计算机返回的结果进行矫正，那么这种模式在Clojure出现后对Java程序来说是一个大大的改进，由于时间和篇幅的限制，用Clojure来编写Java应用程序的细节在此不一一赘述，具体的细节请大家参加黑客道的培训。这样才能获得更多的详细情况和系统的训练。总之利用Clojure让广大的Java程序员编写出安全的，高效的支持并发的，以函数式编程的方式去开发Java应用程序。对于应用程序开发是一个巨大的利益。现在我们对于运行在Java虚拟机上的语言Scala来做如下叙述。

 Scala语言将函数式编程的风格和面向对象编程结合的很好，Twitter公司当年为了提高服务器的效率把他们用Ruby语言开发的程序用Scala语言替换掉了，这说明在性能上Scala比原来的Ruby要好，这个特点也在于JVM给它带来性能上的提升，前面讲过java提供了多线程的支持，对于高并发，高负载的应用提供Java NIO这样的支持，这些特性都被Scala的设计者所采用，Scala语言和Java语言的互操作是非常好的。我发现凡是工作在JVM上的语言和Java语言的互操作都是非常流畅的，只是互操作的形式在语法上有差异而已。这点请大家毋庸置疑。

现在我们来看第三种工作在JVM上的Groovy语言，Groovy语言发明的一个很大用途就是构建脚本，什么叫做构建脚本呢？在写了C，C++程序后，需要使用Makefile脚本，把所有文件编译得到一个可执行文件。在Java的社团里面曾经有很多的技术，最初的Actor利用XML语言来写构建系统，后来的Maven构建系统，现在有了Groovy语言以后，这些语言显得有些过时。

实际上在现在的Java语言的构建系统中Groovy语言的流行度在急剧上升。原来的构建系统技术我认为只有Makefile能够留下来，它留下来的场合也仅仅用于大规模的Java应用程序构建脚本。这些应用程序才会使用Makefile。而其他的，新的Java应用程序的开发，它的构建系统会越来越多的使用Groovy语言来进行编写，当然了Groovy语言可以让Java语言变得动态这也是非常有吸引力的，也就是说Groovy语言可以当做Java语言的脚本来用。这一点在工程实践上是非常有吸引力的。

现在我们来看看第四种工作在JVM上的语言Javascript，Javascript语言曾经被Larry Wall先生推荐为最流行的一种编程语言，倡导大家学习和使用。每一个现代的浏览器都有Javascript的实现。曾几何时Java和Javascript的互操作性不是那么流畅，java8中发布的javascript引擎Nashorn,这个javascript是构建在JVM之上的，因此他们的互操作性很好，可以用javascript为java应用程序写脚本。由于javascript在所有客户端都已经内置了，所以在web应用程序的开发上面，他们的互操作会有更多新的创意。这一点请广大的创客们引起足够的重视。

从我刚才介绍的Clojure，Scala,Groovy，Javascript语言来看，现在工作在JVM上的语言不止Java一种，而且新诞生的语言和原来的Java程序互操作非常好。这就意味着Java的许多类库可以被重用。对于提高创客道的成员去开发新的创业项目是极有吸引力的。从这些年计算机工业的飞速发展，新的进展中可以看到，黑客道中讲解计算的本质，编译原理。编译原理作为计算的本质，它的应用的价值是十分巨大的。尽管计算机发展飞速，但是内核却没有变化。那么从Java虚拟机诞生的这些语言来说，编译技术在其中发挥了巨大的作用。而要理解编译原理，对于黑客道初段的课程要有精准的理解。理解并能熟练的应用到编译原理和编译器的构造中去，这样才能以不变应万变。

毫无疑问，Java技术不是计算机工业发展的终点。将来还会有更强大的技术出现，将来新技术的进展我们必须本着编译原理，计算的本质的理解来看待，这样才能够从新技术中获得利益。关于java语言，Java语言相关的技术以及创客道学员的学习暂且说到这里。前面我说过Java语言的语法形式是C系列。也就是说学过C或C++的学员，再去学习和使用Java的话，相对来说难度降低。那么实际上在历史上C语言的扩充和变形，除了C++和Java以外，还有其他的一些常识，那么在下一讲我会讲一个非常有趣的常识，即Go语言。
