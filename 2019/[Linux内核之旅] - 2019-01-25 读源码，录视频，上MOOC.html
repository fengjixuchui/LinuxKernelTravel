读源码，录视频，上MOOC怎么回事

      是这样的，陈老师一直在带“Linux内核分析与应用”这门课，学校要求课程上MOOC，但陈老师怕麻烦，就随课堂录了《Linux操作系统与原理》一书的全程视频，大概2000分钟左右，那其中的每一讲大概50分钟，但MOOC要的是短小精悍，于是学校就与“学堂在线”联系让陈老师来录MOOC。不干不知道，一干吓一跳，MOOC制作起来的确不容易，陈老师在这次MOOC制作中，对内核的知识重新进行了梳理，每一张图片的选取都花费大量的时间，也再次感受到内核的强大和有趣，与20年前初次涉及Linux内核不同，这一次，有点撤出来看内核的感觉了，也是再次感到，这么一个宝藏，应该有更多的人参与进来挖掘？你，可以么？

你将有什么受益？

1. 感受教师角色。你可能觉得自己某些代码读懂了，但是试试讲出来，不讲不知道，一讲吓一跳，原来自己是一知半解啊，那怎么办？你必须把相关边边角角的知识都拉出来反复琢磨，不断地去悟，结果你会发现，知识不是学出来的，原来是悟出来的，因为某一点悟透了，结果把相关的很多东西都打通了，于是你对业余时间当名教师产生了浓厚的兴趣。

2. 陈老师指导。你录制好了，提交上来了，陈老师或者大家一看，很多知识点理解有误啊，还有细节不到位啊，于是提建议，你在反复修改中发现自己的问题，在修正bug中不断成长。

3. 上MOOC，对，你录制的MOOC提交后，自己觉得很满意，大家也觉得满意，你录制的内容就可以与陈老师的MOOC一块上学堂在线，让更多的人从中受益了。一个的价值在于什么？在于惠及更多的人，这是陈老师20年前写Linux内核书时初步感悟，这20年来一路走来深切感悟的。

4. 在Linux内核之旅发布。即使你录制的视频没有上MOOC，但你已经初步踏入教师的行列了，Linux内核之旅平台将进行发布，让更多的人受益。

录制内容有什么范围么？

有，但也没有。

以《Linux操作系统原理与应用》为主，但不限于这本书（这次MOOC录制内容超出这本书），参考《深入理解Linux内核》，《Linux内核设计与实现》，《深入Linux内核架构》等，武特博客（http://edsionte.com/techblog）上的内容可以作为很好的参考，但这个博客上的内核版本比较早了，需要你在新的内核版本下重新阅读源码或者调试内核。以下是《Linux内核分析与应用》视频录制问卷，你可以进去选择自己感兴趣的部分录制。



录制时需要哪些注意事项？

以下是参与视频录制的同学给出MOOC录制基本流程和注意事项：


前期准备工作

前期准备主要包括以下内容：

一是内容选择，主要是确定所录制linux内核的模块，例如进程部分、内存管理部分、文件系统部分等，并且要明确注明内核版本号，例如Linux-4.19.10；

二是如果需要，可以根据所讲内容制作相应多媒体课件，版面设计清晰简洁，无不相关内容的存在；

三是确保录制现场无噪音，最好带话筒进行录，符合录制要求。


视频录制

1、教材，教学辅助工具及资料准备

（1）所有教学工具、资料尽量不要出现个人信息，在文件夹、文件的命名上尽量和所讲内容相关，例如helloworld.c。

（2）将需要在录制中使用的书籍、资料置于方便自己拿到的地方（避免录制中需要参阅而中断录制，影响录制质量）。

（3）将手机等通讯工具置于静音状态，防止产生噪音影响录制质量。

2、录制时的注意事项

（1）自我介绍：录制开始前1分钟，可以做一个简短的自我介绍。

（2）在录制中需要说明所录制的linux内核模块和版本号。

（3）整体录制时长应控制在15分钟左右（不超过20分钟）。

（4）授课过程尽量使用普通话，咬字清晰，无明显口音口误。

（5）在录制过程中尽量不重复，不说口头禅。

（6）推荐一次性录制，如遇到讲述错误，可等待1-2秒，之后重新讲述，待录制完毕后使用视频剪辑软件剪辑即可。

（7）录制视频桌面壁纸与Linux相关。

3、录制后生产文件的注意事项

（1）视频文件格式要求是mp4格式。

（2）视频文件无需压缩，推荐1080p。

（3）视频文件的命名格式为：模块名称-内容-录制者姓名.mp4，例如：实战演示-Linux内核模块的插入和删除-张三.mp4


4. 录制的视频放在哪

（1）大家可以自行上传到自己熟悉的视频网站，这里建议大家使用哔哩哔哩（没有广告）。

（2）制作好的视频请将链接放到如下的github仓库中，https://github.com/xylinuxer/mooc ，参与讨论评选。
